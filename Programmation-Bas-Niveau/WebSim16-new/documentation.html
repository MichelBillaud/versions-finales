Le processeur SIM16

est une machine fictive destinée à illustrer les notions de base de la programmation au niveau des instructions de la machine.

Sa structure a été calquée sur les premiers ordinateurs : SSEM de l'université Victoria de Manchester, PDP/1 de Digital Equipement Corporation, etc.

Architecture

Le processeur Sim16 a une architecture très simple
  • mots de 16 bits, adresses 12 bits
  • nombres entiers en complément à 2
  • un accumulateur A (16 bits)
  • une mémoire de 128 mots, théoriquement extensible à 4K mots
  • un registre CI (current instruction) de 12 bit contient l'adresse de l'instruction à exécuter (compteur de programme, dans la terminologie actuelle)
  • additionneur/soustracteur 16 bits
  • un bit AUTO pour le fonctionnement (0 si arrété)
  • un répertoire de 13 instructions seulement.
  • modes d'adressages : immédiat, direct, indirect.

Jeu d'instruction

Les instructions, codées sur 16 bits, ont toutes le même format :
OP S (opérande)
o o o o s s s s s s s s s s s s
  • les bits 15-12 contiennent le code opération sur 4 bits
  • les bits 0-11 contiennent un opérande S
Le tableau ci-dessous récapitule les 12 instructions et les mnémoniques que nous utilisons. Selon l'instruction, S est
  • soit une valeur immédiate imm12
  • soit une adresse directe adr12
  • soit une adresse indirecte ind12
M désigne la mémoire, considérée comme un tableau de mots. A désigne l'accumulateur.
Mémonique Signification Effet
loadi imm12 charge dans A la valeur immédiate imm12, avec extension de signe sur 16 bits. A = ext16(imm12) ; CI++
load dir12 charge dans A le contenu du mot d'adresse dir12 A = M[dir12] ; CI++
loadx ind12 charge dans A le contenu du mot pointé par le mot d'adresse ind12 A = M[M[ind12]] ; CI++
store dir12 copie le contenu de A dans le mot d'adresse dir12 M[dir12] = A ; CI++
storex ind12 copie le contenu de A dans le mot pointé par le mot d'adresse ind12 M[M[ind12]] = A ; CI++
add dir12 ajoute à A le contenu du mot d'adresse dir12 A += M[dir12] ; CI+
sub dir12 soustrait de A le contenu du mot d'adresse dir12 A -= M[dir12] ; CI+
jmp adr12 saute à l'adresse ard12 CI = adr12
jneg adr12 saute à l'adresse adr12 si A est négatif si A < 0 alors CI = adr12 sinon CI++
jzero adr12 saute à l'adresse adr12 si A est nul si A == 0 alors CI = adr12 sinon CI++
jmpx ind12 saute à l'adresse contenue dans le mot d'adresse ind12 CI = M[ind12]
call adr12 appel de sous programme à l'adresse adr12 M[adr12] = CI+1 ; CI = adr12+1
halt 0 arrêt auto = 0

Le langage d'assemblage

Le langage d'assemblage permet de rédiger les programmes, instruction par instruction, en utilisant des mnémoniques. Exemple 1 
# exemple
debut
    load a   
    add  b   
    sub  c   
    store d  
    halt 0   
# variables
a  word 42   
b  word  12
c  word 3
d  word 0
  • Il y a une instruction par ligne.
  • les commentaires commencent par un dièse, et finissent au bout de ls ligne
  • une ligne peut comporter une étiquette qui commence en colonne 1
  • les étiquettes peuvent apparaître seules
  • le mnémonique est séparé de l'étiquette éventuelle et de l'opérande par un ou plusieurs espaces
  • le mnémonique "word" permet de réserver un mot en mémoire, en lui affectant une valeur initiale
Les étiquettes évitent d'avoir à compter les instructions. Sans elles, il faudrait écrire
Exemple 2 
# exemple
  load 5    # 0 
  add  6    # 1
  sub  7    # 2
  store 8   # 3
  halt 0    # 4
# variables
  word 42   # 5
  word  12  # 6
  word 3    # 7
  word 0    # 8
Compléments
  • les majuscules et minuscules sont indifférenciées
  • les nombres peuvent étre écrits en hexadécimal. Par exemple 0x11 équivaut à 17.

Le simulateur

Le simulateur permet
  • L'exécution de l'instruction désignée par CI (step)
  • la modification de la valeur de CI
  • le lancement d'une boucle d'exécution (RUN) en indiquant le délai entre deux instructions
  • L'exécution s'arrête quand l'instruction HALT fait passer l'indicateur AUTO à 0, ou quand une erreur est détectée (tentative d'accès en dehors des limites de la mémoire)
  • L'interruption de cette boucle (PAUSE)
